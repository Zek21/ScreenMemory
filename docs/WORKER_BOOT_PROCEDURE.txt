SKYNET WORKER BOOT — EXACT PROVEN PROCEDURE
===========================================
Updated 2026-03-20. Chevron dropdown method v2.0.0.
Workers are opened via the "New Chat" dropdown chevron on the orchestrator
window, then configured to Copilot CLI mode. The chevron is located
dynamically via UIA scan (button named "New Chat" with width ~16px,
positioned right of the main "New Chat (Ctrl+N)" plus button).

Worktree-based boot (INCIDENT 018) is DEPRECATED and REMOVED.

PREREQUISITES
-------------
- Orchestrator HWND known (for focus restore after dispatch)
- Backend running on port 8420
- GOD Console running on port 8421
- VS Code setting: github.copilot.chat.cli.isolationOption.enabled = true
  in both user settings and workspace .vscode/settings.json

GRID POSITIONS (right monitor, 930x500, taskbar-safe)
-----------------------------------------------------
Alpha:  top-left      (1930, 20)
Beta:   top-right     (2870, 20)
Gamma:  bottom-left   (1930, 540)
Delta:  bottom-right  (2870, 540)

DO ONE WORKER AT A TIME. VERIFY BEFORE OPENING NEXT.

===============================================================================
STEP 1 — OPEN WINDOW VIA CHEVRON DROPDOWN
===============================================================================

The chevron dropdown is the small triangle (width ~16px) to the RIGHT of
the main "New Chat (Ctrl+N)" button in the orchestrator title bar.

Detection method (preferred):
  UIA scan the orchestrator window for buttons with Name="New Chat"
  and width <= 20px. This is the chevron. Use its center coordinates.

Fallback (if UIA fails):
  Get orchestrator window rect via GetWindowRect.
  The chevron is at approximately (rect.left + 192, rect.top + 30).

Procedure:
  1. Focus orchestrator: SetForegroundWindow(ORCH_HWND)
  2. Click chevron: pyautogui.click(chevron_x, chevron_y)
  3. Navigate menu:
     pyautogui.press('down')  # 1: New Chat
     pyautogui.press('down')  # 2: New Chat Editor
     pyautogui.press('down')  # 3: New Chat Window  <-- target
     pyautogui.press('enter')
  4. Wait 3s for window to open

===============================================================================
STEP 2 — FIND THE NEW WINDOW HWND
===============================================================================

Enumerate all visible windows matching "Code - Insiders" in title.
Exclude orchestrator HWND and all existing worker/consultant HWNDs.
The remaining window is the new worker.

Poll up to 25s if needed (window may take a moment to appear).

===============================================================================
STEP 3 — POSITION IN GRID
===============================================================================

Move the new window to its worker slot:

  u32.MoveWindow(HWND, gx, gy, 930, 500, True)

===============================================================================
STEP 4 — SET SESSION TARGET TO COPILOT CLI (with verification)
===============================================================================

Method A (UIA-based, preferred):
  1. UIA scan the worker window for buttons
  2. Find a button whose name contains "Local", "Copilot CLI", or "Cloud"
     These are session-target buttons at the bottom of the chat panel
  3. If found, click it; then type "cli" + Enter to select Copilot CLI

Method B (relative coordinates, fallback):
  1. Click at (gx + 55, gy + 484) -- bottom-left session target area
  2. Wait 1.5s for dropdown
  3. Type "cli" + Enter

VERIFICATION (MANDATORY):
  After setting CLI, re-scan via UIA for session_target field.
  If session_target != "copilot_cli", retry once (press Escape first
  to dismiss any stale quickpick, then repeat Method A or B).

Result:
- Session target = Copilot CLI
- Model auto-switches to Claude Opus 4.6 (fast mode)

===============================================================================
STEP 5 — SET PERMISSIONS (SKIP IN CLI MODE)
===============================================================================

In Copilot CLI mode, there is NO permissions button.
The step is skipped entirely.

If for any reason the worker is NOT in Copilot CLI mode and shows a
permissions button, run guard_bypass.ps1 TWICE:

  powershell -File tools\guard_bypass.ps1 -Hwnd HWND
  powershell -File tools\guard_bypass.ps1 -Hwnd HWND

===============================================================================
STEP 6 — DISPATCH IDENTITY PROMPT
===============================================================================

Uses pyautogui (hardware-level input) for reliable submission.
ghost_type/PostMessage does NOT trigger submission in Copilot CLI windows.

Method A (UIA-based input, preferred):
  1. UIA scan for an Edit control or input area in the worker window
  2. Click it, paste prompt, press Enter

Method B (relative coordinates, fallback):
  1. Click input area at (gx + 465, gy + 415)
  2. Ctrl+V to paste the identity prompt
  3. Press Enter to submit

For subsequent prompts, if Enter stops working, click the Send button
at (gx + 880, gy + 453) instead.

The canonical prompt comes from tools/skynet_invocation.py when available.
Submission must transition the worker from IDLE to PROCESSING.

===============================================================================
STEP 7 — WAIT AND VERIFY
===============================================================================

Verification requirements:
- Worker window is still alive via IsWindow(HWND)
- Worker title contains the worker name (e.g. "You are ALPHA")
- Worker posts identity_ack to the bus (check within 60s)
- UIA state transitions from IDLE to PROCESSING after dispatch

PARTIAL PASS: alive + title_ok is sufficient to proceed.
bus_ack may arrive later if the invocation prompt is large.

Only after verification passes may the next worker be opened.

===============================================================================
KEY FACTS
===============================================================================

1. The chevron dropdown is located via UIA, NOT hardcoded coordinates.
   Button Name="New Chat", width ~16px, right of the plus button.
2. Setting Copilot CLI auto-sets the model to Claude Opus 4.6 (fast mode).
3. Copilot CLI mode has NO permissions button -- step 5 is skipped.
4. github.copilot.chat.cli.isolationOption.enabled MUST be true.
5. pyautogui (hardware-level input) is REQUIRED for all interactions.
   PostMessage/SendKeys/ghost_type do NOT work with Copilot CLI windows.
6. One worker at a time. Verify before opening the next.
7. Worktree-based boot is DEPRECATED (removed in v2.0.0).

===============================================================================
COORDINATE CONSTANTS
===============================================================================

All coordinates are RELATIVE to the worker window origin (gx, gy):

| Constant | Offset | Purpose |
|----------|--------|---------|

| Input area | (gx+465, gy+415) | Fallback click before paste |

Fallback: (orch_rect.left + 192, orch_rect.top + 30).
