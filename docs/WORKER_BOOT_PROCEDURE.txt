STEP 2 — FIND THE NEW WINDOW HWND
================================================================================

import ctypes, ctypes.wintypes
u32 = ctypes.windll.user32

known = {ORCH_HWND, 657970, ...any other known HWNDs...}
wins = []

def cb(hwnd, _):
    if u32.IsWindowVisible(hwnd):
        buf = ctypes.create_unicode_buffer(512)
        u32.GetWindowTextW(hwnd, buf, 512)
        t = buf.value
        if 'Code - Insiders' in t and hwnd not in known:
            wins.append((hwnd, t))
    return True

u32.EnumWindows(
    ctypes.WINFUNCTYPE(ctypes.c_bool, ctypes.wintypes.HWND, ctypes.wintypes.LPARAM)(cb),
    0
)

# wins[0][0] is the new HWND

================================================================================
KEY FACTS
================================================================================

- The dropdown chevron at (248, 52) is ABSOLUTE screen coordinates on the
  orchestrator window. If the orchestrator moves, this coordinate changes.

- "Local" dropdown position is RELATIVE to the worker window:
  (gx + 55, gy + 484)

- Input area click position is RELATIVE to the worker window:
  (gx + 465, gy + 415)

- Setting "Copilot CLI" as session target AUTOMATICALLY sets the model to
  Claude Opus 4.6 (fast mode). No separate model change needed.

- guard_bypass.ps1 always needs 2 runs. First sets it, second confirms.

- github.copilot.chat.cli.isolationOption.enabled MUST be true.
  false = FORCES worktree isolation (inverted logic in VS Code).
  true = inline execution (what we want).
  Worker must be CLOSED and REOPENED after changing this setting.

- pyautogui.press('enter') works for submitting pasted content as the
  FIRST prompt in a fresh window. If Enter stops working on subsequent
  prompts, click the Send button at (gx + 880, gy + 453) instead.

- Do NOT use ghost_type_to_worker() for these windows — its Enter key
  via PostMessage/SendKeys does not trigger submission in Copilot CLI.
  Use pyautogui (hardware-level input) for all interactions.

- All workers must be on Copilot CLI mode (NOT Agent, NOT Edits, NOT Local).
  Copilot CLI uses edit/create tools instead of Agent Apply patches, which
  prevents INCIDENT 015 (stuck unappliable patches).
